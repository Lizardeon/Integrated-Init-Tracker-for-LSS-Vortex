# Integrated Init Tracker for LSS Vortex

**Integrated Init Tracker** — это пользовательский скрипт (UserScript) для расширения [Tampermonkey](https://www.tampermonkey.net/), предназначенный для удобного ведения боевых столкновений в веб-приложении [Long Story Short (Vortex)](https://vortex.longstoryshort.app/).

Инструмент добавляет полноценную интерактивную панель трекера инициативы прямо в интерфейс приложения, позволяя Мастеру (DM) мгновенно импортировать игроков со «Стола», автоматически считывать броски инициативы из чата и гибко управлять параметрами NPC.

![](https://github.com/Lizardeon/Integrated-Init-Tracker-for-LSS-Vortex/blob/main/screenshot_2.18.png)

## 🌟 Основные возможности

* **Полная интеграция**: Скрипт бесшовно встраивается в оригинальный интерфейс, добавляя новую вкладку «Трекер Инициативы» в панель инструментов Vortex.
* **Умный старт боя**: При создании нового боя скрипт сканирует карточки игроков на «Столе». Он автоматически ищет последние броски инициативы в логе чата, чтобы составить готовый порядок ходов.
* **Синхронизация PC (игроков)**: Кнопка быстрой синхронизации обновляет актуальные ХП, КД, аватары и статусы игроков прямо со «Стола» в реальном времени, не затрагивая при этом созданных монстров.
* **Резервное копирование (Экспорт/Импорт)**: Бой можно в любой момент сохранить в `.json` файл и загрузить обратно на любой другой сессии или после очистки браузера.
* **Гибкое управление HP монстров**:
  * Быстрый ввод урона и лечения через математические выражения (например, `-12` или `+5`).
  * Полноценная поддержка временных хитов (THP) с автоматическим первоочередным поглощением входящего урона.
  * Цветовая индикация полосы здоровья (зеленый / желтый / красный) в зависимости от критичности состояния существа.
* **Интерактивные статусы и дебаффы**: Возможность в один клик добавлять кастомные эффекты для NPC и удалять их простым нажатием на бадж статуса.
* **Надежность данных**: Состояние боя дублируется в `localStorage` браузера, защищая данные от случайного обновления страницы.

## ⚙️ Установка

1. Установите расширение **Tampermonkey** для вашего браузера ([Chrome](https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmkfjojejmpbldmpobfkfo), [Firefox](https://addons.mozilla.org/firefox/addon/tampermonkey/), [Edge](https://microsoftedge.microsoft.com/addons/detail/tampermonkey/iikmkjmpaadaobahmlepogmconbnnjoc)).
2. Нажмите на ссылку для автоматической установки скрипта:
   👉 [Установить Integrated Init Tracker](https://github.com/Lizardeon/Integrated-Init-Tracker-for-LSS-Vortex/raw/refs/heads/main/Integrated_Init_Tracker.user.js)
3. В открывшейся вкладке Tampermonkey нажмите кнопку **«Установить»** (или «Обновить», если ставили ранее).
4. Откройте или обновите вкладку с вашей комнатой на [Vortex](https://vortex.longstoryshort.app/) — вкладка трекера со значком скрещенных мечей появится автоматически!

## ⚔️ Использование

1. **Начало боя**: Нажмите красную кнопку **«Новый бой»**. Во всплывающем окне отметьте персонажей, которые примут участие в столкновении. Скрипт сам заполнит их инициативу на основе последних бросков из лога.
2. **Добавление NPC**: Заполните форму внизу панели (Имя, Инициатива, HP, КД, Заметки) и нажмите «Добавить». Монстр автоматически займет свое место в списке по значению инициативы.
3. **Управление ходом**: Кнопки **«След. ход»** и **«Назад»** перемещают маркер хода. Счетчик раундов увеличивается автоматически, когда ход возвращается к первому участнику.
4. **Смена параметров «на лету»**: Вы можете кликнуть по любому числовому значению в таблице (Инициатива, КД, имя NPC, текущие/максимальные ХП), чтобы отредактировать их прямо в строке.
5. **Работа со статусами NPC**: Нажмите на круглую кнопку `+` в столбце состояний монстра, введите текст эффекта и нажмите Enter. Чтобы снять статус, просто кликните по нему мышкой.

## 🛠️ Технические детали

* **Стек**: Vanilla JS, HTML5, CSS3.
* **Интеграция**: Использует `MutationObserver` для динамического отслеживания вкладок интерфейса Vortex и встраивания кастомных контейнеров.
* **Изоляция данных**: Скрипт оперирует локальным хранилищем и DOM-деревом на стороне клиента, не отправляя данные на сторонние сервера.
